# Survival Predictions Portal

Survival Predictions Portal is an interactive machine learning portal that brings multiple prediction tasks, datasets, notebooks, and source files into one web interface. It is designed to make data science project work easier to review by combining dataset previews, notebook viewing, prediction forms, and source-code exploration.

## What This Project Demonstrates

- Organizing multiple machine learning tasks in one React application
- Loading and previewing CSV datasets from a controlled backend whitelist
- Viewing Jupyter notebook content inside the browser
- Exploring prediction forms for classification and regression tasks
- Connecting a React frontend with an Express backend API
- Using local font assets so the app can run without relying on remote font CDNs

## Included Prediction Tasks

| Task | Dataset / Notebook | Focus |
| --- | --- | --- |
| Titanic Survival Prediction | `tested.csv`, `Task 1 Titanic_Survival.ipynb` | Classification-style survival prediction using passenger attributes |
| Movie Rating Prediction | `IMDb Movies India.csv`, `Task 2 Movies_Rating_Prediction.ipynb` | Rating prediction using movie metadata |
| Iris Flower Classification | `IRIS.csv`, `Task_3_Iris_Flower.ipynb` | Species classification using sepal and petal measurements |
| Sales Prediction | `advertising.csv`, `Task_4_Sales_Predictions.ipynb` | Sales prediction from advertising spend |

## Project Structure

```text
src/                         React frontend and reusable components
server.ts                    Express backend, task API, file loading, and prediction logic
public/fonts/                Local offline font files
*.csv                        Datasets used by the prediction tasks
*.ipynb                      Jupyter notebooks for the ML tasks
package.json                 Project scripts and dependencies
```

## Backend APIs

```text
GET  /api/tasks                   Returns the available task configuration
GET  /api/file-content/:filename  Returns whitelisted dataset, notebook, or source content
POST /api/predict                 Returns a local prediction result for the selected task
```

## Prediction Logic Note

The portal includes local deterministic prediction logic in the backend so the app can run as a self-contained demo without requiring external model services. The included notebooks and datasets show the related data science work, while the web app focuses on making the tasks interactive and easy to inspect.

For production-grade ML deployment, the next step would be to export trained model artifacts from the notebooks, version them, and load those saved models through the backend.

## Getting Started

Install dependencies:

```bash
npm install
```

Run the development server:

```bash
npm run dev
```

Open the app:

```text
http://localhost:3000
```

Build for production:

```bash
npm run build
```

Start the production build:

```bash
npm start
```

## Limitations

- Prediction outputs are generated by local backend logic for demonstration.
- The app does not call an external ML service.
- Some notebook outputs are trimmed before display to keep the browser experience responsive.
- A production version should load saved trained model files and include reproducible evaluation reports.
